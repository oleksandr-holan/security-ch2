# БЕЗПЕКА ПРОГРАМ ТА ДАНИХ

## Лабораторна робота №9

## Дослідження стійкості парольного захисту

**Мета:** дослідити залежність стійкості парольного захисту архівів від довжини та якості паролів.

**Обладнання:**

* операційна система Windows;
* створений власноруч архів winrar, winzip або інший, захищений паролем;
* програма **Advanced Archive Password Recovery Professional** (можна завантажити за адресою <http://hackerstvo.ru/rar.html>) для розкриття запаролених архівів;
* MS Excel для побудови графіків;
* будь-яке середовище розробки для розробки власної програми брутфорсу.

### Теоретична частина

Встановлення паролів – один з найдавніших та найпопулярніших способів захисту конфіденційності даних. Його використовують для аутентифікації в операційних системах, закриття архівів, розмежування доступу до спільних ресурсів тощо.

Для подолання парольного захисту використовують атаки підбирання паролів, які бувають двох типів: повне перебирання паролів (так званий "брутфорс", атака "грубою силою") та атака за словником, коли програма використовує список відомих найпопулярніших паролів.

Успішність таких атак повністю залежить від довжини та якості встановленого пароля.

Взагалі кажучи, основні вимоги до паролів складаються з наступного:

* Пароль не повинен містити назву облікового запису користувача або фрагменти назви довжиною більше двох символів.
* Довжина пароля не повинна бути меншою за шість символів.
* Пароль повинен містити символи трьох з чотирьох наступних категорій: латинські великі літери (A - Z); латинські малі літери (a - z); цифри (0 - 9); додаткові спеціальні символи (наприклад, !, $, #, %).

Можна також використати як пароль слова російською або українською мовами, набрані при увімкненій латинській розкладці клавіатури. Як правило, такі паролі сприймаються системами контролю якості паролів як сильні.

### Практична частина

1. Для виконання лабораторної роботи створіть архів (.rar або .zip) та захистіть його паролем.
2. Завантажте **Advanced Archive Password Recovery Professional** з вказаної на початку роботи адреси та встановіть її. Там же можна ознайомитися з порядком роботи з цією програмою. Можна використати й іншу аналогічну програму.
3. Як пароль спочатку встановіть одну цифру, наприклад, "1".
4. Запустіть **Advanced Archive Password Recovery Professional** та спробуйте підібрати пароль методом суцільного перебирання. Запам’ятайте час, який програма витратила для цього.
5. Дослідіть залежність часу підбирання від довжини пароля. Для цього збільшуйте його довжину, не змінюючи категорії (тобто використовуйте лише цифри) та кожного разу визначайте час, витрачений програмою для його підбирання. Побудуйте графік залежності часу зламу від довжини пароля для однієї категорії. Зробіть висновок про доцільність використання однієї категорії для створення паролів.
6. Задайте пароль з використанням трьох з чотирьох доступних категорій. Для початку оберіть короткий пароль, наприклад, 3-4 символи. Поступово збільшуючи довжину пароля, дослідіть час, який витрачає зламник на подолання встановленого пароля. Побудуйте графік отриманої залежності.
7. Задайте пароль українським або російським словом, набраним англійською розкладкою, наприклад, слово «пароль» буде виглядати як «gfhjkm» та дослідіть час, необхідний на його підбирання.
8. Порівняйте отримані результати та зробіть висновок про оптимальну стійкість досліджених паролів для заданих обчислювальних ресурсів атакувальника.
9. Повторіть усі дослідження, використовуючи замість повного перебирання паролів атаку за словником. Словники можна завантажити з Інтернету або скласти самостійно. Як змінилися Ваші результати?

    > **ДЛЯ ОТРИМАННЯ ДОДАТКОВИХ БАЛІВ ПРОПОНУЄТЬСЯ РОЗРОБИТИ ВЛАСНУ ПРОГРАМУ БРУТФОРСУ І СЛОВНИКОВИХ АТАК ТА ВИКОРИСТАТИ ЇЇ ДЛЯ ДОСЛІДЖЕНЬ СТІЙКОСТІ ПАРОЛЬНОГО ЗАХИСТУ.**

10. Оформіть звіт з лабораторної роботи.

**Звіт має містити:**

1. Протокол Ваших дій;
2. Графіки досліджених залежностей часу підбирання паролів від їх довжини для усіх випадків.
3. Висновок про найстійкіший з досліджених паролів. Який час потрібний при заданих обчислювальних ресурсах для його зламу?
4. Запропонуйте методи захисту від атак, використаних у цій лабораторній роботі. Аргументуйте їх ефективність.
5. Ті, хто розробляв власний брутфорсер, мають подати блок-схему програми та (на вибір) діаграми класів, модулів або діаграму потоків даних, розроблених в UML-середовищі або Microsoft Visio.
6. Відповіді на контрольні питання.

### Контрольні питання

1. Яку властивість інформації захищають за допомогою паролів?
2. Де переважно використовують парольний захист?
3. Які вимоги висуваються до парольного захисту?
4. Чи підтверджуються ці вимоги результатами Ваших досліджень?
5. Які Ви знаєте способи захисту від атаки повного перебирання паролів та атаки за словником? Чому Ви вважаєте їх ефективними? Які з них використовують в операційних системах та іншому програмному забезпеченні?
6. Які Ви можете запропонувати способи вдосконалення парольного захисту?
